# 权利要求路线C草案（资源调度 + 迭代收敛 + 安全合规）

## 路线定位

- 核心保护点：资源受限调度、迭代收敛停止条件、安全合规联动过滤。  
- 主要规避目标：与“通用多Agent编排”类专利区分。

## 独立权利要求草案（方法）

### 权利要求1（草案）

一种面向双大模型分歧处理的资源调度与收敛控制方法，其特征在于，包括：

1. 获取模型A和模型B对同一问题的输出并识别分歧点；  
2. 根据分歧点数量、分歧置信度差和设备状态计算处理预算；  
3. 按所述处理预算动态分配推理、检索与裁决资源；  
4. 执行分歧解耦与证据裁决，生成当前轮融合结果；  
5. 将关键证据反馈至模型A和模型B并触发下一轮处理；  
6. 当满足分歧阈值、迭代次数阈值或时间阈值中的至少一项时停止迭代；  
7. 在每轮处理前后对输入输出执行合规检测，并对不合规内容进行改写或屏蔽。

## 从属权利要求候选

1. 根据权利要求1的方法，其中预算函数为 `budget = α*|D| + β*conf_gap + γ*risk_level`。  
2. 根据权利要求1的方法，其中设备状态至少包括 CPU 利用率、GPU 显存占用和队列时延。  
3. 根据权利要求1的方法，其中高风险分歧点优先分配证据检索预算。  
4. 根据权利要求1的方法，其中迭代停止条件包括 `|D_t| <= D_min`。  
5. 根据权利要求1的方法，其中迭代停止条件包括迭代次数达到 `【最大迭代次数】`。  
6. 根据权利要求1的方法，其中迭代停止条件包括处理时长达到上限 `T_max`。  
7. 根据权利要求1的方法，其中合规检测至少包括隐私风险检测、歧视风险检测和违法内容检测。  
8. 根据权利要求1的方法，其中每轮迭代输出收敛曲线与资源消耗日志。
